# Модуль авторизации и регистрации

## Реализованное поведение

- Регистрация требует ФИО, ИИН, телефон, должность, компанию, пароль и подтверждение ознакомления с правилами.
- ИИН хранится в зашифрованном виде, а для поиска и уникальности используется `iin_hash`.
- Вход доступен по телефону или ИИН с паролем.
- Заблокированный пользователь не может войти в систему.
- После регистрации пользователь получает роль `User`.
- Доступ в Filament `/admin` разрешен только пользователям с ролью `Admin`, которые не заблокированы.
- Пользователь может редактировать ФИО, телефон, email, должность, компанию и пароль в профиле.
- Изменение профиля фиксируется в журнале действий.
- Администратор может заблокировать и разблокировать пользователя через Filament без удаления его истории.
- Блокировка и разблокировка пользователя фиксируются в журнале действий.
